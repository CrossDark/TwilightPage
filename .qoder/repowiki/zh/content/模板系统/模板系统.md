## 简介
本文件系统性介绍 TwilightPage（基于 TwilightPage）的模板系统：从架构设计、主模板函数 twilightpage-web 的能力与参数、模板配置文件结构，到布局工具函数（如 margin-note 侧注与 full-width 全宽容器）、模板继承与自定义流程、样式扩展与调试技巧，以及与内容处理系统的集成方式。目标是帮助用户从基础使用到高级定制完成端到端掌握。

**更新** 所有核心模块现已增加详细的注释、参数描述和使用示例，显著提升了代码的可维护性和理解性。

## 项目结构
该仓库采用"库包 + 模板"双层结构：
- 库包层（src/）：封装可复用的模板片段与工具函数，形成 twilightpage-web 主模板及子模板（数学、脚注、引用、图表等）。
- 模板层（template/）：提供示例网站内容、构建脚本与样式资源，演示如何使用库包中的模板函数与样式。

## 核心组件
- 主模板 twilightpage-web：负责页面骨架、语言设置、样式注入、头部导航与正文包裹，并串联数学、脚注、引用、图表等子模板。
- 布局工具：margin-note 用于在边注区渲染内容；full-width 提供全宽容器占位；nest-block 实现嵌套缩进块。
- 子模板：
  - 数学模板：统一公式编号与 HTML 渲染角色。
  - 脚注模板：生成脚注引用与边注中的脚注内容。
  - 引用模板：重写方程与标题引用显示。
  - 图表模板：重写图注与图元素，使其适配边注样式。
- 主题变体：abyss 深色主题，支持自动缩进和嵌套内容。

**更新** 所有组件均增加了详细的参数说明和使用示例，提升了代码的可读性和可维护性。

## 详细组件分析

### 主模板 twilightpage-web
- 功能概览
  - 设置语言与文本环境。
  - 注入多个子模板以统一数学、脚注、引用与图表的渲染行为。
  - 构造 HTML 文档骨架：head（meta、title、多条样式表）、body（header 导航、article.main.content）。
  - 支持通过参数自定义标题、语言与样式表列表。
- 关键参数
  - header-links: 导航链接集合，用于生成顶部导航。
  - title: 页面标题。
  - lang: 文档语言。
  - css: 样式表数组，支持 CDN 与本地路径。
  - content: 主体内容占位。
- 处理流程
  - 在 head 中循环输出 css 列表。
  - 在 body 中先渲染 header，再包裹 article.section(content)。
  - 通过 show: 调用各子模板，确保内容在 HTML 目标下被正确包装与标注。

**更新** 主模板现在包含详细的参数描述和使用示例，明确了每个参数的作用和默认值。

### 布局工具：margin-note、full-width 与 nest-block
- margin-note
  - 将传入内容渲染为带 marginnote 类的内联块，适合窄屏时自动转为段落式显示。
  - 可用于插入图片或简短说明，配合脚注高亮联动。
- full-width
  - 将传入内容放入带 fullwidth 类的容器，实现视觉上的全宽布局。
- nest-block
  - 根据标题层级创建嵌套缩进效果，带有左侧边框线。
  - 支持自定义起始深度、边框颜色和缩进距离。

**更新** 所有布局工具函数都增加了详细的参数说明、使用示例和最佳实践指导。

## 性能考量
- 编译阶段
  - 合理拆分子模板，避免在主模板中做重型逻辑，有助于增量编译与缓存命中。
  - 控制样式表数量与体积，优先使用 CDN 与按需加载策略。
- 运行阶段
  - 响应式样式在窄屏下将边注转为块级，减少复杂定位计算，提升首屏渲染效率。
  - 数学渲染采用轻量框架包装，避免过度 DOM 结构。

**更新** 性能考量现在包含了更详细的优化建议和最佳实践。

## 故障排查指南
- 页面未显示边注
  - 检查是否在 HTML 目标下渲染（子模板仅在 target() == "html" 时生效）。
  - 确认 margin-note 的类名与样式文件一致。
- 脚注跳转异常
  - 核对脚注编号与锚点 id 是否匹配；确认引用与脚注内容的 id 生成逻辑一致。
- 数学公式显示异常
  - 确认数学模板已注入；检查 role 与 figure/span 的包装是否正确。
- 构建失败
  - 确保 typst 已安装且版本满足要求；检查 template/Makefile 的编译命令与路径。
  - 若使用本地包，请确认 link 目标指向正确的版本目录。
- 参数配置错误
  - 检查参数类型和默认值是否正确；确认参数传递顺序和语法格式。

**更新** 故障排查现在包含了参数配置相关的常见问题和解决方案。

## 结论
TwilightPage 的模板系统以 twilightpage-web 为核心，通过子模板模块化实现数学、脚注、引用、图表的统一渲染；借助布局工具提供边注、全宽容器和嵌套缩进功能；通过配置文件实现继承与覆盖，结合 Makefile 实现自动化构建。该体系兼顾易用性与可扩展性，既适合快速起步，也支持深度定制。

**更新** 所有核心模块现已具备详细的注释、参数描述和使用示例，显著提升了系统的可维护性和理解性，为用户提供了更好的开发体验。
